Smart Buildings:

In this section we discussed 3 topics which were "what does smart mean", "Smart building scenarios" and "Needed features".
Our combined solutions were that "smart" means a way to automate tasks while removing the humans out of the processing equation to reduce errors. Smart building scenario can be lighting - detecting and motion and turning on or off lights accordingly. key card access which can then turn on certain applications that the particular user has predetermined to be automated upon entry. The features that are required to achieve such samrt building are as following:

  • Sensor detection/inputs
  • Data collection/storage
  • learn behavioural patterns
  • Processing power
  • output(perform required task)

LoT Interfaces

For the interfaces portion we discussed the "features", "users" and "parts needed". These three topics really helped fine tune my understanding of LoT and how it is applicable/used in every day life.
The high level features are the ability and monitor the smart building, such as the CO2 levels, gas, light, tempetature(inside and outside), sun/rain and wind levels.
With all this data collected we can really achive a very smart home that will be able to reduce living and or operating costs. From there we continued to discuss the users that the smart home might be interacting with. These are most likely to be the public entering the building, security guards, staff, students, emergency services and many more. The last part for the control interfaces was the parts needed to create such building. For this we agreed on the following:

  • Sensors
  • Data storage(servers)
  • Processing computer(local or remote)
  • Code/Scripts
  • Humans to monitor building operation and functionality
